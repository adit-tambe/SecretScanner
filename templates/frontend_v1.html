Rapid

The basics for CI/CD & quick local checks

5MB
Max File Size
  • Git Operation: --depth 1 --single-branch
  • Max File Size: 5,242,880 bytes (5MB cutoff)
  • Web BFS Crawl: Capped at 20 URIs
  • Git Coverage: HEAD pointer only. Misses reverted secrets.
  • I/O Strategy: Direct RAM read < 1MB, mmap fallback up to 5MB
  • Complexity: O(F) where F = current tree

Standard

Advanced inspection for daily developer workflows

20MB
Max File Size
  • Git Operation: --depth 100 (Full branch traversal)
  • Max File Size: 20,971,520 bytes (20MB cutoff)
  • Web BFS Crawl: Capped at 50 URIs
  • Git Coverage: Traverses last 100 commits. Catches recently deleted/reverted secrets.
  • I/O Strategy: Aggressive mmap chunking for medium monoliths
  • Complexity: O(F + C_{100})

Deep

Thorough security audits & compliance

∞
Max File Size
  • Git Operation: Full Git Database Object Clone
  • Max File Size: 0 (Unrestricted IO buffer)

  • Git Coverage: Complete traversal of all branches, tags, and orphaned commits.
  • I/O Strategy: Uncapped mmap limits (Requires high RAM for massive blobs)
  • Complexity: O(F_{all} * Commits)
